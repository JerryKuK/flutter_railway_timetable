## ADDED Requirements

### Requirement: 顯示查詢路線標題
時刻表頁面 SHALL 在頁面頂部顯示出發站 → 到達站與查詢日期。

#### Scenario: 標題顯示正確路線資訊
- **WHEN** 使用者從首頁導航至時刻表頁面
- **THEN** 頁面顯示「{出發站} → {到達站}」與查詢日期（格式 YYYY/MM/DD (週X)）

### Requirement: 顯示班次列表
時刻表頁面 SHALL 顯示班次卡片列表，每張卡片包含車次名稱與編號、出發時間、抵達時間、行駛時間、票價。

#### Scenario: 顯示班次卡片
- **WHEN** API 回傳班次資料時
- **THEN** 頁面列出所有班次卡片，依出發時間遞增排序

#### Scenario: 班次卡片內容
- **WHEN** 顯示某班次卡片時
- **THEN** 卡片顯示車次名稱（如「自強 3000」）＋編號（如「#171」）、出發時間、抵達時間、行駛時間（格式 Xh Xm）、票價（格式 NT$ XXX）

### Requirement: 載入中狀態
時刻表頁面 SHALL 在 API 請求進行中顯示載入指示器。

#### Scenario: 資料載入中
- **WHEN** 時刻表頁面正在向 API 請求資料時
- **THEN** 頁面顯示載入指示器，班次列表不可見

### Requirement: 空狀態畫面
時刻表頁面 SHALL 在 API 回傳無班次時顯示空狀態畫面，含火車圖示、「暫無班次資料」說明文字與「重新查詢」按鈕。

#### Scenario: 無班次資料
- **WHEN** API 回傳班次列表為空時
- **THEN** 頁面顯示空狀態畫面，包含火車圖示、「暫無班次資料」標題、說明文字及「重新查詢」按鈕

#### Scenario: 點擊重新查詢
- **WHEN** 使用者在空狀態畫面點擊「重新查詢」
- **THEN** 系統返回首頁讓使用者修改查詢條件

### Requirement: 錯誤狀態
時刻表頁面 SHALL 在 API 請求失敗時顯示錯誤訊息。

#### Scenario: API 請求失敗
- **WHEN** 網路錯誤或 API 回傳錯誤時
- **THEN** 頁面顯示錯誤訊息與重試按鈕
